想像一下，如果今天你想為自己的商家或社群打造一個自動回覆的 LINE 機器人，但想到要寫複雜的程式碼、設定安全的金鑰，甚至上傳程式碼到 GitHub 備份時，每次都要手動複製貼上那串長到像火星文一樣的 Token 密鑰，是不是就讓你大傷腦筋？

過去，這確實需要手動反覆複製貼上。但在這一次的「二部曲」實驗中，我與 AI 協同開發助理 Antigravity 再次攜手合作，我們不僅在短短 15 分鐘內成功把 LINE 機器人升級為**搭載 Google 企業級 Vertex AI (Gemini 2.5 Flash) 的智慧多模態助手**，還一鼓作氣解決了惱人的「GitHub Token 自動化認證」與「遠端倉庫無痛遷移」難關！

這篇文章不談生硬冰冷的程式碼，而是用最白話的「人機協作」視角，帶你一窺現代開發實況，看看我們是如何一一攻克這些難關，實現真正的「免密碼自動通關」！

用生活例子看懂「大腦升級與新配備」
為了讓機器人從「學舌鳥」變成「會看、會思考」的智慧管家，Antigravity 幫我挑選並設定了這幾樣全新升級裝備：

1. Vertex AI Gemini 2.5 Flash（超級智慧大腦）：Google 的新一代 AI 旗艦模型。不僅反應速度極快，還具備「多模態」能力，也就是能同時看懂文字與圖片。
2. 在地化生活指南（System Instruction）：我們強迫大腦在思考時，必須遵循流暢、有人情味的「台灣繁體中文」回覆，避免出現冷冰冰或非在地的簡體用語。
3. 專職行李搬運工（MessagingApiBlobClient）：在最新版的 LINE 設計圖（v9 SDK）中，原本負責傳送訊息的接待員變輕量了，拿不動沉重的圖片檔案。因此，我們必須請出這位專門搬運圖片、影片等二進位大行李的「搬運工」。

第一關：大腦在台灣迷路了？（404 區域限制事件）
我們很順利地改好了程式碼，將其部署上位於台灣（asia-east1）的 Google 雲端伺服器。

我滿懷期待地在 LINE 傳送了第一句問候，沒想到，大腦傳回了冷冰冰的英文錯誤訊息：
"Publisher Model ... was not found or your project does not have access to it."

白話翻譯：「找不到大腦！或是您的專案不被允許使用它！」

AI 助理的即時偵錯與自動修復
看到 404 錯誤，我本來以為是權限又設定錯了。但 AI 助理 Antigravity 迅速分析後，用很生動的比喻告訴我：
「這是因為 `gemini-2.5-flash` 這款最新型的大腦，目前在 Google 台灣伺服器（asia-east1）還沒有正式開通。它找不到這個模型！」

那該怎麼辦？難道要把整個機器人遷出台灣嗎？

Antigravity 提出了一個非常聰明的兩全其美策略：
「我們讓 LINE 機器人（前台接待）留在台灣，這樣與 LINE 伺服器對接的速度最快；但是，當需要思考時，我們讓它把訊息送到完全支援新大腦的美國（us-central1）區域！」

僅僅花費 5 秒鐘，AI 助理在雲端幫我把 `GCP_LOCATION` 環境變數改為 `us-central1`。一測試 — — 機器人立刻用極其流利的台灣繁體中文向我問好！大腦，接通了！

第二關：裝上眼睛！「這不是我要的搬運工！」
接下來是重頭戲：傳送照片分析。

當我興奮地在 LINE 傳送了一張牛肉麵的照片時，機器人又報錯了：
"client.getMessageContent is not a function"

白話翻譯：「接待員說：我根本沒有『下載照片』的這個功能！」

AI 助理的技術除錯
這時候就展現出 AI 協同開發的最強優勢了。如果是一般工程師，可能要翻閱幾十頁的官方 API 更新日誌。
但 Antigravity 在 3 秒內就查明了原因：
「在 LINE SDK v9 之後，『下載照片』這個重體力活已經從普通接待員（MessagingApiClient）身上卸載了。我們必須實例化專屬的搬運工（MessagingApiBlobClient），並且把下載參數從『一個物件袋子』改成『直接給照片識別證 (messageId)』！」

AI 助理一邊解釋，一邊流暢地改寫了程式碼，實例化專屬搬運工。遇到照片時，搬運工把照片下載下來並包裝成 Base64（電子掃描檔），無縫遞交給 Gemini 進行看圖分析。

重新部署後，奇蹟發生了！我傳了一張照片，機器人幾秒內就回覆：
「📸 收到照片，正在幫您分析... 這是一碗熱騰騰的台灣紅燒牛肉麵！湯頭濃郁，牛肉燉煮得非常入味，麵條看起來十分 Q 彈...」

它真的看懂了！而且是用最親切、完全沒有「簡體味」的台灣繁體中文！

第三關：倉庫搬家與「免密碼自動通關」設定
大腦與雙眼都搞定後，我想把這套具備超能力的設計圖（程式碼）備份到我的新 GitHub 倉庫（https://github.com/zonawang/line-ai-bot.git）。

這時候，我們遇到了許多人在使用 GitHub 時最頭痛的「認證難關」 — — 每次上傳程式碼（Push）都要求輸入密碼，而且自從 GitHub 取消密碼登入後，必須使用長得像亂碼的「個人訪問令牌（Personal Access Token）」，每次都要手動複製貼上，真的會讓人懷疑人生。

更糟糕的是，因為我們的上傳是在背景環境中執行，Git 無法彈出視窗讓我輸入金鑰，導致上傳任務直接卡死。

這一次，我直接把我的 GitHub Token（ghp_...）交給了 AI 助理。

Antigravity 展現了非常專業的資安思維與工程技巧。它提出：我們可以直接把 Token「安全地嵌入在本地的 Git 遠端網址中」：
`https://ghp_xxx@github.com/zonawang/line-ai-bot.git`

這樣做的好處是：從今以後，不論是我還是 AI 助理，在這台電腦的此專案目錄下執行任何 Git 上傳（Push），系統都會自動利用這把鑰匙通關，再也不需要手動輸入任何驗證碼或 Token 密鑰，實現真正的「一鍵免密碼上傳」！

貼心附錄：如何 3 步驟申請專屬的 GitHub Token？
如果你也想讓你的專案實現免密碼上傳，這裡由 AI 助理整理了最簡單的「3 步驟申請指南」：

1. 進入 GitHub 設定：登入 GitHub 後，點選右上角頭像 -> Settings -> 拉到最下方的 Developer settings -> Personal access tokens -> Tokens (classic)。
2. 生成新 Token：點選 Generate new token (classic)。在 Note 欄位輸入這個 Token 的用途（例如 "line-bot-deploy"），過期時間（Expiration）可以選擇 30、60 天或永久，並勾選「repo」權限（這能允許我們讀寫儲存庫）。
3. 複製並保存：滑到最下方點擊 Generate token。請務必「立刻複製」畫面上顯示的那串以 `ghp_` 開頭的超長代碼！因為一旦重新整理網頁，這串代碼就再也看不到了。

拿到之後，只要讓 AI 助理幫你設定好遠端網址，就能像我一樣，享受免手輸的金鑰自動化通關體驗！

結語：當技術障礙被整平，創意便能無限奔馳
回顧這整趟升級旅程：
1. 我們讓機器人有了思考大腦（Vertex AI）
2. 為大腦融入了在地文化與語氣（System Instruction）
3. 為它裝上了看世界的雙眼（LINE Blob + 多模態分析）
4. 搞定了繁雜的倉庫搬遷與 GitHub 金鑰自動化認證

這一切，如果全憑個人摸索，在面對 404 錯誤、SDK 版本 API 異動、以及終端機認證卡住時，可能要花上幾天甚至放棄。但在 AI 助理的精準執行與白話解釋下，每一次的錯誤都成了我們 1 分鐘內快速通關的養分。

未來的軟體開發，重點不再是記憶那些生硬的指令，而是「如何清晰地表達你的創意與邏輯」。20 分鐘，一個搭載最新多模態 AI、能自動備份的 LINE Bot 就此誕生。

不論你是不是工程師，我都非常鼓勵你試著與 AI 助手合作，把腦中的好點子動手實作出來。科技不該是高牆，而是一扇只要你願意對話，就會為你開啟的門。

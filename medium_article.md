## 🛠️ 第一關：大腦在台灣迷路了？（404 區域限制事件）

我們很順利地改好了程式碼，將其部署上原有的台灣（asia-east1）Google 雲端伺服器。

我滿懷期待地在 LINE 傳送了第一句問候，沒想到，大腦傳回了冷冰冰的英文錯誤訊息：
> ❌ *[VertexAI.ClientError]: Publisher Model ... was not found or your project does not have access to it.*

白話翻譯：「找不到大腦！或是您的專案不被允許使用它！」

### 💡 AI 助理的即時偵錯與跨時區繞道
看到 404 錯誤，我本來以為是權限又設定錯了。但 AI 助理 Antigravity 迅速分析後，用很生動的比喻告訴我：
> 「這是因為 `gemini-2.5-flash` 這款最新型的大腦，目前在 Google 台灣伺服器（asia-east1）還沒有正式開通。它找不到這個模型！」

那該怎麼辦？難道要把整個機器人遷出台灣嗎？這樣連線速度不就變慢了？

Antigravity 提出了一個非常聰明的兩全其美策略：
> 「我們讓 LINE 機器人（前台接待）留在台灣，這樣與 LINE 伺服器對接的速度最快；但是，當需要思考時，我們讓它把訊息送到完全支援新大腦的美國（us-central1）區域！」

僅僅花費 5 秒鐘，AI 助理在雲端幫我把 `GCP_LOCATION` 環境變數改為 `us-central1`。一測試 — — 機器人立刻用極其流利的台灣繁體中文向我問好！大腦，接通了！

## 📸 第二關：裝上雙眼！「這不是我要的搬運工！」

接下來是這次升級的重頭戲：讓機器人看懂照片。

當我興奮地在 LINE 傳送了一張牛肉麵的照片時，機器人又報錯了：
> ❌ *TypeError: client.getMessageContent is not a function*

白話翻譯：「接待員說：我根本沒有『下載照片』的這個功能！」

### 💡 舊接待員拿不動大行李：LINE SDK v9 的 API 異動
這時候就展現出 AI 協同開發的最強優勢了。如果是一般工程師，可能要翻閱幾十頁的官方 API 更新日誌。
但 Antigravity 在 3 秒內就查明了原因：
> 「在 LINE SDK v9 之後，『下載照片』這個重體力活已經從普通接待員（`MessagingApiClient`）身上卸載了。我們必須實例化專屬的搬運工（`MessagingApiBlobClient`），並且把下載參數從『一個物件袋子』改成『直接給照片識別證 (messageId)』！」

AI 助理一邊解釋，一邊流暢地改寫了程式碼，實例化專屬搬運工。遇到照片時，搬運工把照片下載下來並包裝成 Base64（電子掃描檔），無縫遞交給 Gemini 進行看圖分析。

重新部署後，奇蹟發生了！我傳了一張照片，機器人幾秒內就回覆：
> **「📸 收到照片，正在幫您分析... 這是一碗熱騰騰的台灣紅燒牛肉麵！湯頭濃郁，牛肉燉煮得非常入味，麵條看起來十分 Q 彈...」**

它真的看懂了！而且是用最親切、完全沒有「簡體味」的台灣繁體中文！

## 📋 貼心附錄：如何 3 步驟申請專屬的 GitHub Token？

如果你也想讓你的專案實現免密碼上傳，這裡由 AI 助理整理了最簡單的「3 步驟申請指南」：

1. **進入 GitHub 設定**：登入 GitHub 後，點選右上角頭像 -> **Settings** -> 拉到最下方的 **Developer settings** -> **Personal access tokens** -> **Tokens (classic)**。
2. **生成新 Token**：點選 **Generate new token (classic)**。在 Note 欄位輸入這個 Token 的用途（例如 "line-bot-deploy"），過期時間（Expiration）可以選擇 30、60 天或永久，並勾選「**repo**」權限（這能允許我們讀寫儲存庫）。
3. **複製並保存**：滑到最下方點擊 **Generate token**。請務必「立刻複製」畫面上顯示的那串以 `ghp_` 開頭的超長代碼！因為一旦重新整理網頁，這串代碼就再也看不到了。

拿到之後，只要讓 AI 助理幫你設定好遠端網址，就能像我一樣，享受免手輸的金鑰自動化通關體驗！
